Mantenimiento del Sistema

A medida que se vaya prolongando en el tiempo el empleo del Simulador Táctico Galeón, la Base de Datos del sistema puede ser actualizada o ampliada por el usuario al introducir nuevos equipos, unidades, juegos, etc.. Por tanto, con el paso del tiempo la Base de Datos puede evolucionar, enriquecerse, y permitirá cada vez un mayor número de posibilidades para la simulación de distintas situaciones tácticas.

Por otro lado, las grabaciones de Ejercicios, tanto en modo continuo como en modo fotografía, proporcionada por el Logger que incorpora el sistema, permite conservar un histórico de los mismos, permitiendo así, de alguna manera, almacenar la experiencia adquirida.

Por tanto, ya que son elementos que evolucionan con la experiencia y que dan valor añadido al Simulador Táctico Galeón, se recomienda prestar una especial atención al mantenimiento de la Base de Datos y de las Grabaciones de Ejercicios generadas. Este mantenimiento consistirá en el almacenamiento fuera del sistema de estos datos, de forma que sólo será necesario conservar instalado en el sistema exclusivamente lo que esté en uso en cada momento, ya que podremos recuperarlos si es necesario. Esto permitirá además mantener el sistema limpio de datos obsoletos.

Para realizar este mantenimiento, se hace necesario un mecanismo de copias de seguridad de estos datos. En este apartado se incluyen instrucciones y recomendaciones para realizar copias de seguridad de la Base de Datos y de las Grabaciones de Ejercicios.

Instrucciones para realizar Copias de Seguridad de la Base de Datos:

Para hacer copias de seguridad de la Base de Datos del Simulador es necesario un mecanismo que permita el paso a fichero de los datos. Este mecanismo se implementa mediante dos herramientas del sistema de gestión de la base de datos denominadas Export e Import.

  • Export hace una copia de los datos y el esquema de los mismos en un archivo del sistema operativo.

  • Import lee los archivos creados al exportar y coloca los datos y su esquema en el sistema de gestión de base de datos.

Estas herramientas son accesibles desde la Herramienta de Preparación del Simulador Táctico Galeón (ver Manual de Preparación – Descripción General).

Las copias de seguridad deben hacerse de los ficheros de exportaciones.

Este mecanismo es el que debe emplearse también para realizar una réplica de una base de datos sobre otro usuario, exportando la base de datos del usuario que se desea replicar, e importándola con el nuevo usuario. Ver el apartado Anexos – Administración de la BD.

Instrucciones para realizar Copias de Seguridad de las Grabaciones de Ejercicios:

Como se explica en el apartado Manual de Análisis - Grabación de Ejercicios, los ejercicios pueden ser grabados en modo continuo o en modo fotografía. Tanto los ficheros correspondientes a las grabaciones en modo fotografía como todos los ficheros en subdirectorios correspondientes a una grabación continua, son almacenados en el directorio al que apunta el parámetro LoggerPath configurado en el Galeon.ini.

Las grabaciones en modo fotografía pueden realizarse desde cualquier Consola de Instructor, y la grabación continua es realizada bien por el servidor cuando se trata de una ejecución en red de un ejercicio como por cualquier consola si se trata de una ejecución local. Por tanto, para mantener los ficheros de grabaciones en un único sitio, se recomienda configurar el parámetro LoggerPath apuntando a un directorio compartido por la red. De este modo, al estar todas las grabaciones en un único sitio se facilita su gestión y uso (cargarlos desde cualquier consola, clasificarlos en directorios, borrado de ficheros obsoletos, etc.).

Si se sigue esta recomendación, las copias de seguridad de las grabaciones deberán hacerse simplemente del contenido de este directorio.

Recomendaciones de Soporte y Periodicidad para las Copias de Seguridad:

El soporte recomendado para realizar las Copias de Seguridad, por su capacidad y facilidad de uso es CD-R y CD-RW.

Se recomienda hacer copias de seguridad en un ciclo mensual como se explica:

  • Seleccionar un día fijo de la semana para hacer copias de seguridad.
  • Hacer copias semanales en CD-RW reutilizando el soporte cada cuatro semanas.
  • Hacer copia de seguridad permanente en CD-R al final de cada ciclo mensual.

Recomendaciones adicionales:

En periodos en los que se considere que se estén generando muchos datos nuevos semanalmente, ya sea en Base de Datos como de Grabaciones de Ejercicios, se recomienda aumentar la periodicidad de las copias de seguridad, intentando siempre seguir un patrón parecido al recomendado. Por ejemplo, realizando dos o tres copias NO PERMANENTES semanales en vez de una y dos copias PERMANENTES mensuales.

En aquellas ocasiones en las que se considere que puntualmente se han generado datos de especial importancia o interés, ya sea en Base de Datos como de Grabaciones de Ejercicios, se recomienda realizar copia de seguridad PERMANENTE adicional.
